# SueTogether Platform — 기능 요구사항 정의서

> SMA Protocol Phase 1 산출물

---

## 1. 시스템 개요

차세대 B2B SaaS 기반 집단분쟁 및 다중지역소송(MDL) 초자동화 플랫폼.
사건의 발굴(Lead Generation)부터 대화형 접수, 비정형 증거 분석, 소송 결과 예측,
최종 합의금 분배까지 소송의 전 생애주기를 관장.

## 2. 비즈니스 모델

- **B2B SaaS**: 법무법인이 월 구독료 지불
- **다중 테넌트 격리**: 엄격한 RLS + RBAC
- **보안**: PIPA(개인정보보호법) 완벽 준수
- **고부하**: 수십만 명 동시 접속 대응

## 3. 핵심 기능 모듈

### Module A: Intake Service (인테이크 서비스)

| 기능 | 설명 | 우선순위 |
|------|------|----------|
| AI 챗봇 | 24시간 자연어 기반 원고 자격 필터링 | P0 |
| 본인인증 | KCB/NICE/PASS API 연동 실명 검증 | P0 |
| 대규모 중재 엔진 | 수천명 데이터 자동 병합 + 중재기관 양식 패키징 | P1 |
| 리드 제너레이션 | 잠재 원고 트래킹 및 자격 스코어링 | P1 |

#### Service Flow:
```
원고 접속 → 챗봇 대화 시작 → 기본 정보 수집 →
본인인증 API 호출 → 자격 필터링 → 증거 업로드 안내 →
DB 저장 → 실무자 알림
```

### Module B: Document AI Service (증거 검증 서비스)

| 기능 | 설명 | 우선순위 |
|------|------|----------|
| VLM 파이프라인 | LayoutLMv3/Donut 기반 비정형 문서 구조화 | P0 |
| 교차검증 | 입력 데이터 vs VLM 판독 데이터 실시간 대조 | P0 |
| 교정 피드백 | 불일치 시 자동 재업로드 요청 반환 | P1 |
| S3 스토리지 | 안전한 증거 파일 저장/관리 | P0 |

#### Service Flow:
```
파일 업로드 → S3 저장 → BullMQ 큐 등록 →
VLM 워커 처리 → 구조화 데이터 추출 →
교차검증 실행 → 일치/불일치 판정 →
불일치 시 교정 피드백 반환
```

### Module C: RAG Service (증거 지식화 서비스)

| 기능 | 설명 | 우선순위 |
|------|------|----------|
| 폐쇄형 RAG | 업로드된 증거 DB 내에서만 검색 증강 생성 | P0 |
| 의료 연대기 | 의료 기록에서 진단/처방 시간순 추출 | P0 |
| 딥링크 매핑 | AI 생성물 → 원본 PDF 페이지 링크 | P0 |
| 할루시네이션 통제 | 출처 없는 정보 생성 차단 | P0 |

#### Service Flow:
```
문서 업로드 → 텍스트 추출 → 청크 분할 →
벡터 임베딩 → pgvector 저장 →
질의 시 RAG 검색 → 출처 포함 응답 생성 →
원본 PDF 페이지 딥링크 첨부
```

### Module D: Analytics Service (예측 분석 서비스)

| 기능 | 설명 | 우선순위 |
|------|------|----------|
| 판결 예측 | 과거 판례 + 판사 성향 기반 승소/기각 확률 | P1 |
| CBA 대시보드 | 재판 진행 vs 합의 비용-편익 분석 | P1 |
| GIS 핫스팟 | 오픈데이터 + 원고 주소 지도 시각화 | P1 |
| 피해 밀집도 | 구조적 피해 지역 자동 탐지 | P2 |

#### Service Flow:
```
사건 데이터 입력 → 판례 DB 검색 → 판사 성향 매칭 →
ML 모델 예측 → CBA 계산 →
원고 주소 지오코딩 → GIS 레이어 중첩 →
핫스팟 탐지 → 대시보드 시각화
```

### Module E: Settlement Service (합의금 분배 서비스)

| 기능 | 설명 | 우선순위 |
|------|------|----------|
| 전자서명 | 합의금 내역 확인 + 디지털 서명 | P0 |
| 계좌 실명조회 | 은행 API로 예금주 검증 | P0 |
| PG 결제 연동 | 토스/나이스 착수금/인지대 결제 | P0 |
| 펌뱅킹 대량이체 | 수천명 자동 송금 워커 | P0 |
| 세금 자동계산 | 원천징수/부가세 자동 공제 | P1 |

#### Service Flow:
```
합의 성립 → 에스크로 입금 확인 →
원고별 분배금 계산 (수수료, 세금 공제) →
전자서명 요청 → 서명 완료 확인 →
계좌 실명조회 → 펌뱅킹 대량이체 큐 등록 →
배치 이체 실행 → 개별 결과 추적 → 완료 알림
```

## 4. 공통 기능 (Core Module)

| 기능 | 설명 |
|------|------|
| 멀티테넌트 미들웨어 | 요청 헤더에서 tenantId 추출, RLS 적용 |
| RBAC 인가 | 역할별 API 접근 제어 (5단계) |
| PII 암호화 | AES-256-GCM 컬럼 레벨 암호화 |
| 감사 로그 | 모든 민감 작업 불변 기록 |
| 봇 탐지 | reCAPTCHA + 브라우저 핑거프린팅 |
| Rate Limiting | IP/사용자/테넌트 레벨 속도 제한 |
